# 드롭 프로젝트 — 팀 안내서 (제일 먼저 읽기)

## 한 줄로

**"눈 가린 우체부"** 를 만든다. 크리에이터가 콘텐츠를 자물쇠로 잠가 올리고, 구매자가 프라이버시 결제(누가 냈는지 안 보이는 Zcash)를 하면, 중개 서버가 **열쇠를 배달**해 준다. 근데 이 서버는 특수 하드웨어 안에 있어서 **콘텐츠도 못 보고 누가 샀는지도 모른다.** 서버 주인조차 못 들여다본다.

```
크리에이터                  [눈 가린 우체부 = 서버]                   구매자(폰)
   │ ① 콘텐츠 자물쇠 채워 올림        (특수 하드웨어 TEE 안,         │
   │ ② 콘텐츠 열쇠를 우체부한테만      아무도 못 봄)                  │ ③ 프라이버시 결제 +
   │    몰래 넣어줌 ───────────▶  ┌──────────────┐                  │   "내 임시열쇠"를 쪽지에
   │                             │ ④ 결제 감지   │◀──── 쪽지(메모)───┤
   │                             │ ⑤ 열쇠 포장   │                   │
   │                             └──────┬───────┘                   │
   │                                    │ 포장한 열쇠를 게시판에 올림 │
   │                                    └──────────────────────────▶ ⑥ 받아서 자물쇠 풀고 콘텐츠 봄
```

## 우리가 검증한 것 (스파이크 3개 — 전부 통과 ✅)

**스파이크** = 다 만들기 전에 "이거 진짜 될까?" 싶은 위험한 부분만 미리 빨리 실험한 것.

| | 질문 | 어떻게 | 결과 |
|---|---|---|---|
| **#1** | 구매자 지갑이 결제에 "쪽지(메모)"를 같이 보내주나? | 폰 Zashi로 QR 스캔→실제 결제 | ✅ 쪽지 그대로 체인에 실림 |
| **#2** | 서버가 "보기 전용 권한(IVK)"만으로 결제 감지 + 쪽지 읽기 가능? | probe 프로그램으로 실제 거래 읽음 | ✅ 금액+쪽지 복원 |
| **#3** | 콘텐츠 열쇠를 하드웨어 안에만 넣고 서버 주인도 못 보게? | 진짜 Phala 하드웨어에 띄워 검증 | ✅ 안에서만 풀림, 진짜 Intel 인증 |

**→ 가장 위험했던 3개가 다 통과 → "이거 진짜 만들 수 있다" 확정.**

발견한 함정 하나(C6): 지갑이 *최신 버전* 거래를 만들면 우리 라이브러리(구버전)가 못 읽음 → 그 이후 결제를 다 놓칠 뻔. 우회로 고침. (자세한 건 `../feasibility-review.md` §3.6)

## 4명으로 나누기

시스템은 4덩어리. **2 Rust + 2 web** 기준:

| 레인 | 만드는 것 | 누구 | 상세 스펙 |
|---|---|---|---|
| **A1** | 서버 속 엔진: 결제 감지→열쇠 포장→배달 | Rust #1 | [`lane-A1-payment-engine.md`](./lane-A1-payment-engine.md) |
| **A2** | 서버 껍데기: 비밀 받기(봉인)+배포+인증 | Rust #2 | [`lane-A2-enclave-platform.md`](./lane-A2-enclave-platform.md) |
| **B** | 구매자 화면: QR→결제→열쇠 받아 콘텐츠 열기 | Web #1 | [`lane-B-buyer-app.md`](./lane-B-buyer-app.md) |
| **C** | 크리에이터 화면: 콘텐츠 암호화 올리기+서버 검증 | Web #2 | [`lane-C-creator-app.md`](./lane-C-creator-app.md) |

**딱 하나 규칙:** 첫날 1시간 모여서 [`interfaces.md`](./interfaces.md)(서로 주고받는 데이터 6개 모양)만 확정하고 흩어진다. 그거 정하면 각자 "남의 부품을 가짜로 대체"하고 혼자 끝까지 만들 수 있다 → 마지막에 붙여도 딱 맞음.

**오늘 당장:** Rust #1 = A1 설계도대로 시작 / Rust #2 + Web #2 = 비밀 넣기(#3 확장) / Web #1 = 구매자 화면.

## 어려운 말 사전

| 말 | 쉬운 뜻 |
|---|---|
| 스파이크 | 본격 개발 전 핵심만 빨리 해보는 실험 |
| TEE | 안을 아무도(서버 주인도) 못 보는 밀봉된 하드웨어 방 |
| Phala | 그 TEE 방을 빌려주는 클라우드 |
| attestation(인증) | "나 진짜 그 코드 그대로 돌고 있음"을 하드웨어가 증명서로 보여주는 것 |
| 메모(memo) | Zcash 결제에 붙이는 512바이트 쪽지 |
| IVK (보기전용 키) | 들어온 결제를 *볼 수만* 있고 돈은 못 빼는 열쇠 |
| K_drop | 콘텐츠를 잠근 마스터 열쇠(AES) |
| e_pub / e_priv | 구매자가 결제마다 새로 만드는 일회용 키쌍 |
| dispatch blob | K_drop을 e_pub로 포장한 작은 꾸러미(게시판에 올림) |
| lightwalletd | Zcash 체인을 가볍게 읽게 해주는 서버 |
| sealed box | 상대 공개키로만 열 수 있게 포장하는 표준 암호 방식(libsodium) |
| NU / branch | Zcash 네트워크 업그레이드 버전. 라이브러리가 뒤처지면 새 거래 못 읽음(C6) |
| mock(가짜 부품) | 아직 없는 남의 부품을 가짜로 대체해 혼자 개발하는 것 |
| TDD | 테스트 먼저 쓰고 그담 코드 짜는 방식 |
